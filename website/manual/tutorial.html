Overall

STAR browser system is an integrated platform for visualization of next generation sequence(NGS) data. The system provides ability to handle and visualize huge amounts of NGS data.

 

Get started

Register account->log into system

After log into the system, user can browse all public data tracks (accessible by all) and private tracks (only accessible by owner). User can also log into system by "guest login" option, which means only public data tracks are accessible, and the configurations could disappear due to refreshed web browser cookies.

 

Browse tracks

All available tracks are shown in upper right table. Using below options to search desired tracks:

"All tracks" button: all available tracks that can be accessed by this user.

"SDEC tracks" button: specific data sets for UCSD Epigenomic Roadmap project.

"My Tracks" button: all tracks created or owned by this user.

Search by keywords: perform fulltext searching by input keywords(and/or logical operations can be used).

"Tracks" panel in left tab panel: search matched tracks by combined options such as cell, center, experiment type, histone marks and organism.

 

Sort tracks

Click table title to sort tracks by this column.

 

Remove selected tracks

Mark checkbox associated with selected tracks-> click "Remove selected" at the bottom of the table region

 

Edit track information

Only tracks owned by this user can be modified. Track information will be displayed in "Editor" tab panel in right bottom region.

Search tracks->change tabpanel to "Editor"->click "Add to track pool/Editor"/Drag to editor"->Modify-> click "Save Track":

 

Browse configurations. All available configurations are shown in "Configs" tabpanel.

Search configuration: input keywords  in "search input box" and enter.

Sort configuration: click column head titles

Remove configuration: marks checkbox associated with  configurations, and then click "Remove selected".

View configuration: click "View" button associated with the configuration. All the tracks in the configuration will be visualized.

Edit configuration: click "Edit" button associated with the configuration. Track information will be loaded in "Track pool" table

 

Create/Update configuration

 

Create Configuration:  Browse tracks->Mark selected tracks->Select track pool->Drag to Track pool/click "Add to track pool/Editor"->Drag to reorder tracks/Remove selected tracks->change configuration name/description->click "Create configuration"

 

Update Configuration:  Click "Edit" in "Configs" list->Drag to reorder tracks/Add new tracks/Remove selected tracks->change configuration name/description->click "Update" button to overwite and update this configuration

 

Upload new data tracks

Track Meta information:  

Access permission:

Private: tracks can only be seen by  this user

Group: If user specifies a group account during registration, tracks will be accessible to all users belong to the group account

Public: tracks will be released to all users

 

Track type:

Short reads: This type is usually used to visualize aligned ChIP-Seq/RNA-Seq data, such as Eland, Bowtie, BED files, etc. The signals will be displayed as histogram style at higher zooming ratio, while nucletides will be displayed at lower zooming ratio.

 

CG/CHG/CHH: It is used to display methylation sites called from Methyl-Seq/MeDIP-Seq/RRBS/MRE-Seq experiments. A typical result file usually like this:

chr1       469     -       CG      14      15

chr1       471     +      CHG    15      15

chr1       484     -       CHH    16      18

 

Intensity:  It's used to visualize sequence data and real values.The signals will be displayed as histogram style at all zooming levels.

 

Organism:

This is related to proper gene model information, such as chromosomes, genome length, etc. 

 

Institute:

Input organization information. It's helpful when searching  desired tracks.

 

Data source:

Data can be directly upload to STAR server, or distributed in users local machine(remote server). If data is processed and stored in local machine (see server setup section), only data fetchers are requried to provide data track information:

Examples for track name:

bing_T2_eln.heli

bing_T2_input.heli

ryancms_T2_eln.heli

ryancms_T2_input.heli

 

Examples for track URL:

http://tabit.ucsd.edu/fetchers/xue/bing_T2_eln.heli.php

http://tabit.ucsd.edu/fetchers/xue/bing_T2_input.heli.php

http://tabit.ucsd.edu/fetchers/xue/ryancms_T2_eln.heli.php

http://tabit.ucsd.edu/fetchers/xue/ryancms_T2_input.heli.php

 

Upload data files:

Prepare data files and provide format information for direct upload.

This module requires Adobe flash plugin. If you can not see 'upload' button, please check whether your web browser blocks flash plugin.

Supported file format: BED, WIG, Eland, Bowtie, Mapview, SAM, GFF, self defined

If self defined is selected, users need to input a string to specify which columns should be extracted from tab-delimited text file. The string looks like: chromosome index :start index :end index :strand index : value index : sequence index, and each index number is separated by ':'. For example: 1:2:3:4:6:-1 means chromosome, start position, end position, strand and value will be extracted from column 1, 2, 3, 4 and 6, respectively, and there's no sequence information.

Compressed files supported: .TAR, .GZ, .TAR.GZ, .ZIP

File format is also examined on server side. If file does not pass the format check, it will be ignored.

 

After data files are uploaded to server, or data fether information is provided, click 'Submit' to complete. Users should see those tracks in the track list.

 

Setup remote data

To help building up local server and data processing tools.

Please download  star-browser.tar.gz and follow instructions:

-------------------------------------------------------------------------------- 

Requirements:

* GNU C++

* Berkeley DB

--------------------------------------------------------------------------------

1. Install Berkeley DB(Oracle)

(1) tar xzvf db-4.6.18.tar.gz

(2) cd  db-4.6.18/build_unix/ (Select appropriate directory based on OS type)

(3) ../dist/configure --prefix=/usr/local/berkeleydb/ --enable-cxx   (BDB install directory)

(4) make

(5) make install

(6) copy shared libs to system directories

copy /usr/local/berkeleydb/lib/*.so /usr/lib/

copy /usr/local/berkeleydb/lib/*.so /usr/lib64/

 

--------------------------------------------------------------------------------

2. Install STAR package

Suppose all above packages have been installed and been configured successfully,

The installation instructions of this package will be:

cd src &

vim Makefile

 

CXXFLAGS += -I/usr/local/berkeleydb/include/

LDFLAGS := -L/usr/local/berkeleydb/lib/ -ldb_cxx -lpthread

 

Modify above two lines if you have different BDB installation directory.

 

make &

make install &

You will see three program:

data processing: bin/ngsproc,

mini web server: www/websrv,

CGI program for data fetcher: www/cgi-bin/fetcher.cgi

 

--------------------------------------------------------------------------------

3. Start web server

(1) Modify www/cgi-bin/fetcher.conf

    List all directories which contain you processed data by ngsproc

(2) ./websrv &

    Mini web server will be started and monitor on port 8080 (do not let firewall block this port)

 

--------------------------------------------------------------------------------

4. Process mapped NGS data

Usage:./bin/ngsproc [options ....]

-h --help Display this usage information

-a --action      database operation, in [create|query|drop|level|count]

-d --db-dir      Database directory

-t --table       Database table you want to operate

-i --input       Input data filename, only for 'create'

-f --format      Data format, in [BED|WIG|BOWTIE|BOWTIESAM|SAM|ELAND|ELANDMULTI|ELANDEXPORT|GFF|MAPVIEW]

-s --start       Start position, only for 'query', default 0

-e --end         End position, only for 'query', default 100000

-c --chrom       Chromsome index, default 1

 

Examples:

(1) ./bin/ngsproc -a create -t H3K4me1_tbl -f bed -d /data/ngs-data -i /usr/tom/H3K4me1.bed

 

Create BDB table from input H3K4me1.bed file. Table name: H3K4me1_tbl; Table path: /data/ngs-data

File format: BED

 

(2) ./bin/ngsproc -a create -t h1.H3K4me3 -f 1:2:3:-1:5:-1 -d /data/ngs-data -i /usr/tom/H3K4me3.txt

Create BDB table from input H3K4me3.txt file. Table name: h1.H3K4me3; Table path: /data/ngs-data

File format: 1:2:3:-1:5:-1, means(chromsome, start, end, strand, value, sequence) will be extracted

from those columns(input file contains many TAB delimited columns). In this example, chromosome from

column 1, start postion from column 2, end position from column 3, value from column 5, and there's no

strand information(regard as positive strand) and sequence information.

(3) ./bin/ngsproc -a query -t H3K4me1_tbl -d /data/ngs-data -s 0 -e 100000 -c 2

Query previously processed reads on chromosome 2, and the region is between 0 and 100000

This can be used to check if your data are properly processed.

 

(4) ./bin/ngsproc -a level -d /data/ngs-data

This will levelize all your data tables(tracks) under this directory. It's very important to speed up

data retrieving, especially when browsing at high zooming level.

--------------------------------------------------------------------------------
